Authors

Mauro Castelli, Roberto Henriques, Luca Manzoni, Sara Silva, Leonardo Trujillo and Leonardo Vanneschi

Decription

GSGP is a free/open source C++ library that provides a robust and efficient implementation of geometric semantic genetic operators.

Installation

The GSGP source code only consists of a header file and a .cc file (that contains a main). To compile the code just execute the following command

g++ -Wall -O0 -g GP.cc -o GP

Running the program requires the execution of the command

./GP -train_file train.txt -test_file test.txt

where, in this example, train.txt and test.txt are the files containing the training and the test instances

Running the program in "test mode" (only available in the last version) to evaluate a model on newly provided instances (USE_TEST_SET flag=1 and expression_file=1 in the configuration file) requires the execution of the command

./GP -test_file test.txt

Input files format

Input files are .txt files where values are separated by a TAB. The first two rows of each file represent the number of independent variables of the problem and the number of instances in the dataset. Each row represents an instance, while each column contains the values of a variable. The last column contains the target values. The library comes with two example input files (training and test files).

NOTE: in test mode the test file must not have the number of instances in the dataset and also the target column must be removed.

Configuration file

The library comes with a configuration file that contains the parameters of the algorithm. It is possible to set the following parameters: 

- population size
- number of generations
- initialization method
    - 0 -> growth method
    - 1 -> full method  
    - 2 -> ramped half and half method
- crossover rate
- mutation rate
- maximum depth of the individual for the initialization
- size of the tournament selection
- accept single node individual in the initial population
    - 0 -> not accepted
    - 1 -> accepted
- mutation step
- number of random constants
- minimum possible value for the random constants 
- maximum possible value for the random constants
- minimization problem
   - 1 -> minimization problem
   - 0 -> maximization problem
- number of random trees (only available in the last version)
- expression_file (load individuals from text file called "individuals.txt". Only available in the last version)
   - 1 -> use text file
   - 0 -> use an initialization method
- test mode (only available in the last version) 
   - 1 -> use the optimal model returned by a previous GP run to evaluate newly instances
   - 0 -> standard evolutionary process

Documentation

The documentation of the library is a Doxygen documentation. The implementation has been done in order to use the library after a very quick reading of the documentation.
